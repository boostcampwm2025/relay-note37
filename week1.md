### 1. 자신의 소통 방식을 AI 통해 알아가기

#### ✅ 지금까지의 대화 방식 요약

| 항목                                    | 설명                                                                                                 |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| 🎯 **목표 지향적**                      | 매 대화에 명확한 목적이 있고, 결과물이 나올 때까지 포기하지 않음.                                    |
| 🔁 **반복·피드백 중심**                 | 지속적으로 피드백을 주며 결과 퀄리티를 높임, 효율성과 정확성을 중시하는 태도가 강함                  |
| 🧠 **메타 인지 강함**                   | 본인의 약점(예: 실수 반복, 명확한 문제 정의 부족)을 스스로 인식하고 개선하려는 태도가 있음.          |
| 📚 **개념 학습 -> 실전 적용 흐름 선호** | 단순히 개념만 설명받기보단 예제 코드, 패턴 적용, 실전 흐름까지 이어지는 방식의 답변을 더 선호합니다. |

#### 🛠️ 개선 해보면 좋을점

| 항목                        | 설명                                                                       |
| --------------------------- | -------------------------------------------------------------------------- |
| 🎯 **요청 정교화 훈련**     | 요청할 때 "배경 + 목적 + 기대 결과"를 최소한의 문장으로 말하는 습관 들이기 |
| 🔍 **의사표현 누락 탐색**   | 내가 쓴 메시지에서 **생략된 전제/배경**이 있는지 스스로 검토하기           |
| 🧠 **모호 표현 줄이기**     | 모호한 표현을 더 구체적으로 풀어보기                                       |
| 🤝 **공감적 리포스팅 연습** | ChatGPT의 답변을 **이해한 대로 요약한 후 수정 요청**                       |

#### 수 : 3사 AI 알고리즘 코딩 테스트

알고리즘 문제를 주어주고 풀이를 작성한 후 AI에게 각자의 코드 평가를 맡겼는데, Claude와 Perplexity는 Perplexity, Claude, GPT로 등수를 매겼다. 그런데 GPT의 경우 조금 문제가 있었다. 서버 오류로 답변이 날아갔다고 생각해 같은 질문을 다시 한 번 돌렸는데, 전과는 완전히 다른 평가 결과가 나와 여러 번 시도해본 결과 매번 평가 결과가 다르게 나온다는 것을 알 수 있었다. 때문에 GPT의 응답 신뢰도에 대해서 의문점이 발생했다. 그래도 4번의 시도 중 2번은 Perplexity, Claude, GPT 1번은 GPT, Claude, Perplexity 1번은 Perplexity, GPT, Claude가 나왔다.
(실제 채점 결과는 Claude > GPT > Perplexity > 과거의 나였다)
